# Secure Data Science Lab

## Ziel
Dieses Repository dokumentiert meinen 30-Tage-Einstieg in Data Science und AI Security.
Jeder Tag enthält reproduzierbare Artefakte, Analysen und dokumentierte Erkenntnisse.

## Struktur
- src/              → Python-Code
- notebooks/        → Explorative Analysen
- data/raw/         → Originaldaten
- data/processed/   → Bereinigte Daten
- security/         → Sicherheitsnotizen & Risikoanalysen
- evidence/         → Tagesnachweise
- outputs/          → Ergebnisse (Plots, Exporte)
- docs/             → Zusatzdokumentation

## Nachweis-Regel (Daily Evidence Rule)
Jeder Tag gilt als abgeschlossen, wenn:
1. Code reproduzierbar läuft
2. Ein Evidence-File (`evidence/day-XX.md`) existiert
3. Eine kurze Interpretation dokumentiert ist
4. Ein Git-Commit erfolgt ist
